## Query 工作流

> 查询和回答问题的标准流程

### 流程步骤

```
Step 1: 理解问题
  → 提取问题中的关键概念
  → 明确问题类型（what/how/why/对比）

Step 2: 定位相关页面
  → 读取 index.md 查找相关页面
  → 或直接搜索wiki目录

Step 3: 读取相关页面
  → 读取 concepts/ 中的概念页
  → 读取 entities/ 中的实体页
  → 读取 summaries/ 中的摘要页

Step 4: 合成答案
  → 组织答案结构
  → 标注引用来源
  → 提供代码示例（如适用）

Step 5: 自动评估并询问存档
  → AI 自动判断答案是否有长期存档价值
  → 判断标准:
    - 是否涉及核心概念或重要原理
    - 是否包含可复用的代码/配置
    - 是否解决了常见疑问
    - 是否有横向对比或深入分析
  → 如果有价值 → 在回答末尾提示: "这个答案涉及 [主题]，我认为有长期价值，是否存档到知识库？"
  → 用户回复"是"或"存档" → 创建 wiki/analysis/[问题主题].md
  → 用户不回复或回复"否" → 不存档
```

### Query Prompt 模板

```markdown
## Query任务

### 问题
[用户问题]

### 执行
1. 读取 index.md 定位相关页面
2. 读取相关概念/实体/摘要页
3. 合成答案并引用来源
4. 自动评估答案价值并询问存档（如适用）

### 输出格式
- 简洁回答（what/how类）
- 分析报告（why/对比类）
- 代码示例（如适用）
- 如果答案有长期价值，在末尾添加提示："这个答案涉及 [主题]，我认为有长期价值，是否存档到知识库？"
```

### 日常问答流程（每日维护参考）

```
Step 1: 理解问题
Step 2: 读取index.md定位相关页面
Step 3: 读取相关页面
Step 4: 合成答案
Step 5: 存档(如有必要)
  → 如果答案有长期价值
  → 创建 wiki/analysis/[主题].md
  → 在相关概念/实体页添加链接
```
