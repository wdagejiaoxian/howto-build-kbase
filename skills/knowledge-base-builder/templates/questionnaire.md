# 问卷模板（8 题完整版）

一次性向用户提出以下问题：

| 序号 | 问题 | 目的 | 示例回答 |
|------|------|------|---------|
| Q1 | 你的职业/身份是什么？ | 确定用户角色 | "Python 后端开发"、"学术研究者"、"产品经理" |
| Q2 | 你主要收集哪个领域的资料？ | 确定技术领域 | "AI/机器学习"、"前端开发"、"市场营销" |
| Q3 | 你收集的资料主要是什么格式？ | 确定支持格式 | "技术博客 markdown"、"PDF 论文"、"网页截图" |
| Q4 | 你大约有多少份资料？ | 预估处理量 | "50 份左右"、"几百份"、"刚开始收集" |
| Q5 | 你希望分几个大类？（可选） | 确定分类粒度 | "3 个：语言/框架/部署"、"按项目分" |
| Q6 | 你平时怎么查找资料？ | 确定检索习惯 | "关键词搜索"、"按标签浏览"、"按时间线" |
| Q7 | 你最希望知识库帮你解决什么问题？ | 确定核心需求 | "快速找到某个 API 用法"、"整理学习笔记" |
| Q8 | 有没有特殊的知识组织习惯？ | 确定个性化需求 | "用标签系统"、"需要代码示例"、"要画图" |

> **注意**：用户不需要全部回答。至少回答 Q1-Q3 即可开始。
