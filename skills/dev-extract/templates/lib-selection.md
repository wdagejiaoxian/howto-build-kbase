# 📦 库/框架选型模板

> ⚠️ 必须达到 L3 原理层

## Frontmatter

```yaml
type: dev-note
subtype: lib-selection
tech_stack: [技术栈列表]
status: 已采用|已替换|已放弃
encounter_count: 1
last_encountered: YYYY-MM-DD
valid_versions: []
expires:
created: YYYY-MM-DD
updated: YYYY-MM-DD
```

## 正文模板

````markdown
# [库/框架名] 选型记录

## 选型场景

[在什么项目/模块中需要选型]

## 候选对比

| 维度     | [库A]    | [库B]    |
| -------- | -------- | -------- |
| 版本     | x.x      | x.x      |
| 性能     | [数据]   | [数据]   |
| 学习曲线 | 低/中/高 | 低/中/高 |

## 最终选择

**[库名] [版本号]**

## 选择理由

1. **[理由1]**: [详细说明，含原理]
2. **[理由2]**: [详细说明]

## 放弃的替代方案

### 为什么没用 [库B]

[具体原因——不是简单说"不满足需求"]

## 原理深挖（L3 必须）

### [选定库的核心架构]

[底层设计思路——为什么这么设计]

### 与其他方案的本质差异

[在架构思想上的根本不同]

## 使用心得

### 踩过的坑

1. [坑1]: [现象 + 原因 + 解决]

### 最佳实践

```python
# 推荐的使用方式
```

## 适用边界

- ✅ 适合: [场景]
- ❌ 不适合: [场景]

## 相关链接

> 根据配置中的 `link_style` 选择以下一种格式，删除其余两种：

**link_style = wiki**（双向链接）：
- [[dev-notes/decisions/相关决策]] - 关联的决策
- [[concepts/相关概念]] - 相关概念说明

**link_style = tag**（标签格式）：
- #dev-note/相关决策 - 关联的决策
- #concept/相关概念 - 相关概念说明

**link_style = text**（纯文本）：
- 关联决策: [决策名] - 关联的决策
- 相关概念: [概念名] - 相关概念说明
````
