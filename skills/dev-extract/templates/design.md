# 🏗️ 架构设计模板

> ⚠️ 必须达到 L3 原理层

## Frontmatter

```yaml
type: dev-note
subtype: design
tech_stack: [技术栈列表]
status: 草案|已实施|已废弃
encounter_count: 1
last_encountered: YYYY-MM-DD
created: YYYY-MM-DD
updated: YYYY-MM-DD
```

## 正文模板

````markdown
# [设计主题]

## 设计目标

[这个架构要解决什么问题]

## 架构图（文字描述）

```
┌──────────┐     ┌──────────┐
│  客户端   │────▶│  API层   │
└──────────┘     └──────────┘
```

## 模块划分

| 模块   | 职责       | 技术选型 | 选型理由 |
| ------ | ---------- | -------- | -------- |
| API 层 | 路由、验证 | FastAPI  | [理由]   |

## 数据流

1. [数据从哪里来，经过什么处理，到哪里去]

## 关键设计决策

### 决策 1: [标题]

- **问题**: [要解决什么]
- **方案**: [采用了什么设计]
- **替代方案**: [考虑过但没选的]
- **原理依据**: [为什么这个设计更好——底层原理]
- **取舍**: [牺牲了什么换来了什么]

## 设计原则

1. [原则1]: 为什么遵循这个原则

## 演进记录

- YYYY-MM-DD: 初版设计
- YYYY-MM-DD: [修改了什么，为什么]

## 相关链接

> 根据配置中的 `link_style` 选择以下一种格式，删除其余两种：

**link_style = wiki**（双向链接）：
- [[dev-notes/decisions/相关决策]] - 关联的决策
- [[dev-notes/libs/相关选型]] - 关联的选型

**link_style = tag**（标签格式）：
- #dev-note/相关决策 - 关联的决策
- #dev-note/相关选型 - 关联的选型

**link_style = text**（纯文本）：
- 关联决策: [决策名] - 关联的决策
- 关联选型: [选型名] - 关联的选型
````
