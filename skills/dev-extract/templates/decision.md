# 🔀 方案决策模板

> ⚠️ 必须达到 L3 原理层

## Frontmatter

```yaml
type: dev-note
subtype: decision
tech_stack: [技术栈列表]
status: 已采纳|已否决|待定
encounter_count: 1
last_encountered: YYYY-MM-DD
created: YYYY-MM-DD
updated: YYYY-MM-DD
```

## 正文模板

```markdown
# [决策主题]

## 背景

[为什么需要做这个决策]

## 备选方案

### 方案 A: [名称]

- **描述**: [方案简述]
- **优点**: [优点列表]
- **缺点**: [缺点列表]
- **适用场景**: [什么情况适合]
- **不适用场景**: [什么情况不适合]

### 方案 B: [名称]

[同上结构]

## 决策矩阵

| 维度     | 权重 | 方案A | 方案B |
| -------- | ---- | ----- | ----- |
| 性能     | 高   | ⭐⭐⭐   | ⭐⭐    |
| 开发成本 | 中   | ⭐⭐⭐⭐  | ⭐⭐⭐   |

## 最终决策

**选择：方案 X**

**核心理由**：[最关键的 1-2 个原因]

**取舍说明**：[放弃了什么，为什么可以接受]

## 原理深挖（L3 必须）

### [方案 X 的核心机制]

[底层是如何工作的——不是怎么用，而是为什么这样设计]

### 横向对比原理

[方案 X 和方案 Y 在底层机制上的根本差异]

### 适用边界

- ✅ 适合: [具体条件]
- ❌ 不适合: [具体条件]
- ⚠️ 注意: [临界情况]

## 后续验证

- [ ] 在 xxx 场景下验证方案可行性

## 相关链接

> 根据配置中的 `link_style` 选择以下一种格式，删除其余两种：

**link_style = wiki**（双向链接）：
- [[concepts/相关概念]] - 相关概念说明
- [[dev-notes/decisions/相关决策]] - 关联的决策

**link_style = tag**（标签格式）：
- #concept/相关概念 - 相关概念说明
- #dev-note/相关决策 - 关联的决策

**link_style = text**（纯文本）：
- 相关概念: [概念名] - 相关概念说明
- 关联决策: [决策名] - 关联的决策
```
